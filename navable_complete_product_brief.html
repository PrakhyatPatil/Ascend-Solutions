What data does the delivery network actually collect?

Delivery partners see a simple in-app prompt when they arrive at or leave a location. Takes under 30 seconds. 5 data points per visit.

Entry accessibilityDoes this entrance have a ramp? Step? Both? Neither? (single tap)
Lift / elevator statusPresent and working / broken / absent — timestamped so we know it's current
Path surface qualitySmooth / uneven / under construction — applies to the route from road to entrance
Parking / obstacle blockageIs the footpath clear? Vehicles / vendors blocking it? (common in Indian cities)
Accessible toilet presentYes / No — for mobility-impaired users on longer journeys

How this appears on the Navable map

Every pinned location gets a simple accessibility score (1–5) derived from the 5 data points. The map pin colour reflects the score.

Green pin — fully accessible Amber pin — partial (e.g. ramp but no lift) Red pin — not accessible Grey pin — no data yet

The AI agent can query this data: "Is the mall entrance accessible?" → agent checks pin score + latest timestamp + specific flags → speaks a clear answer.

Family tracking feature

What it does

Live GPS shared with up to 3 trusted contacts. Family sees user on a map in real time. User can press SOS — sends location + voice clip to all contacts instantly.

Privacy-first

User controls sharing. One tap to pause. Sharing expires after a set journey duration — not always-on surveillance.

Prakhyat's ML brief — exactly what to build

Model: YOLOv8-nano → converted to TensorFlow Lite

Pre-trained on COCO dataset (already knows people, cars, bikes). Fine-tune or just run inference on relevant classes. Runs at ~15–20fps on a mid-range Android phone.

Detect these hazard classes (use COCO + custom):

step / staircasevehicle on footpathpotholewet floor signconstruction barrierperson (crowd)

Output format (JSON per frame):

{ "hazard": "vehicle", "confidence": 0.87, "distance_estimate": "near", "direction": "left", "alert_level": "high" }

alert_level drives audio: "high" = immediate voice alert, "medium" = earcon tone, "low" = silent log. Distance estimated from bounding box size relative to frame.

Hardware vision — Navable Smart Cane (post-hackathon)

V1 clip-on sensor

Ultrasonic sensor clips onto any existing cane. Bluetooth to phone. Extends hazard range to 4m. ₹400 BOM cost.

V2 integrated cane

Built-in ultrasonic + haptic grip vibration. Folds like a standard cane. No screen. Works standalone + syncs to app.

Why cane, not glasses

Canes last decades. No battery anxiety. Already trusted by users. Cultural adoption is zero-friction.
