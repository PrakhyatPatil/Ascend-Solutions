4 core features — what the app actually does
1

Live hazard detection

Phone camera runs Prakhyat's YOLOv8-nano model on-device, continuously. Detects obstacles, steps, vehicles on footpath, construction zones. Speaks alerts instantly — no internet needed for this feature. This is the star of your demo.

On-device TFLiteNo latencyWorks offline
2

AI voice agent — the big mic button

User holds centre mic button, speaks naturally. App sends voice → AWS Transcribe (STT) → Amazon Nova Lite on Bedrock (reasoning) → AWS Polly (voice back). Agent knows your GPS, nearby accessibility pins, and recent hazards. Answers anything: "Is the entrance accessible?", "Find me a ramp nearby", "Call my sister".

Amazon Nova LiteAWS TranscribeAWS Polly
3

Accessibility map with crowd pins

Google Maps SDK with custom overlay. Every rated location shows a coloured pin — green (accessible), amber (partial), red (not accessible). Delivery partners rate locations in 5 taps via a mini-form in their existing delivery app. Data stored in DynamoDB, served via Lambda API to the map.

Google Maps SDKDynamoDBLambda API
4

Family safety tracking

User shares live GPS with up to 3 trusted contacts. Family sees them on a map in real time. One-press SOS sends location + a 5-second voice clip to all contacts via SMS/WhatsApp link. User controls when sharing is active — not always-on.

How AWS stitches it all together
Amazon Nova Lite
(Bedrock)

The brain of the voice agent. Receives: user's spoken question + current GPS + 5 nearest accessibility pins + last 3 hazard detections. Returns a natural language answer. Nova Lite is fast and cheap — perfect for real-time voice.

AWS Transcribe
(STT)

Converts mic audio to text in under 1 second. Supports Hindi + English — critical for Indian users who code-switch mid-sentence ("Yaar, nearest ramp kahan hai?").

AWS Polly
(TTS)

Converts Nova's text response back to natural voice. Use "Kajal" (Hindi neural voice) or "Aditi" for Indian accent. Output plays through earphones immediately.

Lambda + API Gateway

3 functions: (1) handle voice agent pipeline, (2) read/write accessibility pins to DynamoDB, (3) handle SOS trigger. No server to manage — deploy in minutes.

DynamoDB

Two tables: AccessibilityPins (location, score, ratings, timestamp) and Users (profile, trusted contacts). Queried by GPS bounding box for nearby pins.

AWS SNS

SOS feature — sends SMS to trusted contacts with a Google Maps link of the user's live location. Instant, reliable, no app needed on family's end.

AWS Cognito

Phone number OTP login. Works with Indian numbers, no credit card needed from users. Handles sessions.

Google Maps SDK

Map rendering + routing only. Custom marker layer for accessibility pins. Free tier covers the hackathon easily. Route calculation can use Google Directions API filtered by accessible paths.

Exactly how the mic button works — the full pipeline
User holds mic → AWS Transcribe streams audio → Lambda builds context prompt → Nova Lite on Bedrock reasons → AWS Polly speaks response → User hears answer in ~2 sec

The context prompt Lambda sends to Nova Lite looks like this:

You are Navable, a navigation assistant for visually and mobility-impaired users.
User location: 22.7196° N, 75.8577° E (Indore)
Nearby pins: Mall entrance [ramp:yes, lift:working, score:4/5, 2h ago], Bus stop [score:2/5, step:yes]
Recent hazards detected: vehicle on footpath (30s ago, left side)
User asked: "Is the mall entrance accessible?"
Reply in 1–2 short sentences. Speak naturally. Mention distance if known.
